# 三、已有的科研工作基础和已具备的科学研究条件

## 1. 已有的科研工作基础和已具备的科学研究条件

### 1.1 数据准备

#### (1) 仿真环境数据集采集

由于VLA模型训练需要的数据过大并且真实数据采集困难且耗时，本研究第一阶段任务采用sim to real的训练策略，使用Isaac Sim仿真软件搭建与真实实验环境相近的仿真环境，并且仿真环境使用与真实实验环境相同的机械臂采集数据以便后续VTLA模型迁移到真实环境。

**环境布置包括：**
- **机械臂本体**：采用睿尔曼RM75-B-V七自由度机械臂，具有高精度控制特性
- **桌面工作台**：标准化桌面环境，与真实实验室配置一致
- **交互目标物体**：透明立方体（Cube），位置可随机化以增加数据多样性
- **视觉传感系统**：
  - 腕部相机（Wrist Camera）：安装于机械臂末端link_7的camera_link处，分辨率320×240，用于近距离精细操作引导
  - 第三视角相机（World Camera）：固定于场景中，分辨率640×480，提供全局视角观察

**采集的数据包括：**
- **视觉数据**：两个视角的RGB图像，以numpy格式存储
- **机械臂状态数据**：
  - 末端执行器位姿（EE pose）：包含三维位置和四元数姿态
  - 关节角度（Joint positions）：8个关节（含夹爪）的角度值
  - 关节角速度（Joint velocities）：各关节运动速度
- **任务相关数据**：目标位置、物体位置、任务成功标志、末端到目标距离等

**数据采集策略：**
- 采用多环境并行采集技术，同时运行20个隔离仿真环境，显著提高数据采集效率
- 每个episode包含250步仿真，每10步保存一次图像数据
- 目标物体位置随机化：80%概率在核心工作区（X: 0.28~0.35m, Y: -0.10~0.10m），20%概率在外围区域
- 机器人初始姿态随机化：在基础Ready姿态上叠加多维度随机噪声
- 采用RMPFlow运动规划算法实现平滑的轨迹生成
- 成功判定标准：XY平面距离<2cm且末端高度在目标上方23~28cm范围内

#### (2) 真实环境数据采集

基于仿真环境的数据采集经验，我们已搭建完整的真实环境数据采集系统。该系统实现了多传感器同步采集，确保视觉数据与机械臂状态的时间对齐，为后续sim-to-real迁移提供高质量的真实数据支撑。

**硬件配置：**
- **机械臂**：睿尔曼RM系列七自由度协作机械臂，通过以太网（TCP/IP）与上位机通信，控制接口IP: 192.168.33.80，端口: 8080
- **第三视角相机**：DJI Osmo Action运动相机，提供高质量RGB图像，分辨率1280×720，帧率30fps
- **腕部相机**：Intel RealSense深度相机（RGB模式），安装于机械臂末端附近，分辨率1280×720，帧率30fps

**采集的数据包括：**
- **DJI相机图像**：第三视角RGB帧，JPG格式存储，用于全局场景理解
- **RealSense相机图像**：近端视角RGB帧，JPG格式存储，用于精细操作引导
- **机械臂状态**：JSON格式存储，包含：
  - 关节角度（joint）：7个关节的实时角度值
  - 末端位姿（pose）：末端执行器的六自由度位姿（位置+欧拉角）
  - 时间戳：精确到毫秒级的采集时间

**数据采集流程：**
1. 系统启动后自动连接DJI相机、RealSense相机和机械臂控制器
2. 实时预览双相机画面及机械臂状态信息
3. 按空格键开始/暂停录制，每次录制创建带时间戳的独立session文件夹
4. 数据同步存储：每帧同时保存双相机图像和对应的机械臂状态
5. 支持多session连续采集，便于收集不同任务场景的数据

**数据组织结构：**
```
sessions/
└── session_YYYYMMDD_HHMMSS/
    ├── dji/                    # DJI相机图像
    │   ├── 0001.jpg
    │   ├── 0002.jpg
    │   └── ...
    ├── realsense_rgb/          # RealSense相机图像
    │   ├── 0001.jpg
    │   ├── 0002.jpg
    │   └── ...
    └── robot_state/            # 机械臂状态
        ├── 0001.json
        ├── 0002.json
        └── ...
```

### 1.2 模型训练

我们使用已采集的仿真环境数据对模型进行一阶段任务的微调，并建立了完整的训练与推理流程。

#### (1) 模型架构

采用MLA-T（Multi-modal Language-Action Transformer）架构，该架构基于预训练的视觉-语言模型（VLM）进行动作策略学习：

- **视觉编码器**：采用DINO-SigLIP双编码器架构（224×224输入分辨率），提取丰富的视觉语义特征
- **语言模型骨干**：基于LLaMA-2-7B大语言模型，具备强大的多模态理解与推理能力
- **动作解码器**：采用Diffusion扩散模型进行动作序列生成，支持Action Chunking（动作分块）技术

#### (2) 训练策略

**第一阶段：仿真数据微调**

基于Isaac Sim采集的RM75机械臂仿真数据进行模型微调，主要配置如下：

| 参数 | 设置值 |
|------|--------|
| 预训练基座 | prism-dinosiglip-224px+7b |
| 动作维度 | 7（7自由度关节角度） |
| 分布式训练 | 双GPU（torchrun，nproc_per_node=2） |
| 批量大小 | 全局batch_size=8，单卡batch_size=4 |
| 学习率 | 2e-5 |
| 最大训练步数 | 100,000步 |
| 检查点保存间隔 | 1,000步 |
| 冻结策略 | 冻结视觉塔（Vision Tower）和LLM骨干，解冻顶部10层进行微调 |
| Action Tokenizer | 启用，将连续动作离散化为256个token |
| 动作预测窗口 | 15步（future_action_window_size） |
| 数据增强 | 启用图像增强（image_aug） |
| 扩散步数 | 4步（repeated_diffusion_steps） |

**第二阶段：真实数据域适应**

使用真实环境采集的数据进行域适应训练，采用相同的模型架构，主要调整：
- 训练轮数：20 epochs
- 数据源：真实环境双相机数据（DJI + RealSense）
- 离线训练模式：支持W&B离线日志记录

#### (3) 推理部署

已实现基于Flask的VLA推理服务器，支持实时机器人控制：

- **推理精度**：BF16混合精度推理，降低显存占用
- **动作生成**：单次推理生成15步动作序列，实际执行前10步
- **反归一化**：使用训练集统计量（q01/q99分位数）将归一化动作还原为物理增量
- **任务Prompt**：支持自然语言任务指令输入，如"Approach the transparent object, align the gripper center with the object's centroid, and hold still 5cm above it"

#### (4) 训练监控与评估

- **实验追踪**：集成Weights & Biases（W&B）进行训练过程可视化
- **损失监控**：实时记录训练损失（典型收敛值：loss≈0.0256）
- **检查点管理**：定期保存模型检查点，支持断点续训

### 1.3 软硬件实验平台搭建

**硬件平台：**
- 睿尔曼RM75-B-V七自由度协作机械臂（以太网TCP/IP通信）
- DJI Osmo Action运动相机（第三视角，1280×720@30fps）
- Intel RealSense深度相机（腕部/近端视角，RGB模式）
- 高性能GPU工作站：双NVIDIA RTX 4090（共48GB显存），用于仿真渲染与模型训练

**软件平台：**
- **仿真环境**：NVIDIA Isaac Sim（基于Omniverse），支持多环境并行仿真
- **运动规划**：RMPFlow（Riemannian Motion Policy）运动规划框架
- **深度学习框架**：PyTorch 2.x + FSDP分布式训练
- **模型架构**：MLA-T（基于LLaMA-2-7B + DINO-SigLIP视觉编码器）
- **视觉采集**：OpenCV + pyrealsense2 SDK
- **机械臂控制**：睿尔曼Python SDK（rm_robot_interface）
- **实验管理**：Weights & Biases（W&B）训练追踪，Conda环境管理
