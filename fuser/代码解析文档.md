## 1. 相机参数详解

### 1.1 CameraParams 数据类

```python
@dataclass
class CameraParams:
    K: torch.Tensor   # 内参矩阵 (Intrinsic Matrix) 3×3
    R: torch.Tensor   # 旋转矩阵 (Rotation Matrix) 3×3
    t: torch.Tensor   # 平移向量 (Translation Vector) 3×1
```

**内参矩阵 K 的物理含义：**

```
K = [fx,  0, cx]
    [ 0, fy, cy]
    [ 0,  0,  1]
```

- `(fx, fy)`：焦距（像素单位），控制视角和尺度
- `(cx, cy)`：主点坐标，图像平面中心偏移量
- 所有相机均有 **焦距相同** 的特点（fx ≈ fy），说明使用的事标准针孔模型

## 2. 坐标投影函数详解

### 2.1 核心投影流程（以 `project_3d_to_2d_672_rlbench` 为例）

```
输入:  xyz_3d [N, 3]  世界坐标系中的3D点
输出: patch_idx [N, 2]  归一化的patch索引 (row, col)
      valid [N]  有效点掩码
```

**Step 1 — 内参按缩放比例调整（Scale K）**

```python
scale_x = 672 / 224 = 3.0   # 图像被放大到 672×672
scale_y = 672 / 224 = 3.0

K_scaled[0,0] *= scale_x   # fx: 307.72 → 923.15
K_scaled[1,1] *= scale_y   # fy: 307.72 → 923.15
K_scaled[0,2] *= scale_x   # cx: 112 → 336
K_scaled[1,2] *= scale_y   # cy: 112 → 336
```

**Step 2 — 将外参 R, t 转为 相机→世界 的变换**

```python
R_world_to_cam = R.T              # 旋转矩阵转置 = 逆（正交矩阵）
t_world_to_cam = -R_world_to_cam @ t  # 相机在世界中的位置
```

**Step 3 — 3D点变换到相机坐标系**

```python
xyz_cam = xyz_3d @ R_world_to_cam.T + t_world_to_cam   # 世界坐标 → 相机坐标
```

**Step 4 — 投影到像素平面（针孔模型）**

```python
uvw = xyz_cam @ K_scaled.T   # 先用内参变换
z = uvw[..., 2:]             # 深度
xy = uvw[..., :2] / (z + 1e-6)  # 归一化投影 xy = (u/z, v/z)
```

**Step 5 — 计算 Vision Token 对应的 Patch 索引**

```python
total_stride = patch_stride × conv_stride = 14 × 3 = 42  # 最终 stride

row = floor(xy[1] / 42)   # 像素y坐标 → patch行索引
col = floor(xy[0] / 42)   # 像素x坐标 → patch列索引

patch_h = 672 / 42 = 16   # 16×16 = 256 个 patches
patch_w = 672 / 42 = 16
```

**Step 6 — 边界裁剪与掩码**

```python
valid = (z > 0) & (0 ≤ xy[0] < 672) & (0 ≤ xy[1] < 672)

row = clamp(row, 0, 15)   # 防止越界
col = clamp(col, 0, 15)
patch_idx = [row, col]
```

### 2.2 三种相机的投影差异

| 相机 | 原始分辨率 | 目标分辨率 | 总Stride | Patch网格 |
|------|-----------|-----------|---------|----------|
| rlbench_front | 224×224 | 672×672 | 42 | 16×16 |
| franka_right | 480×640 | 672×672 | 42 | 16×16 |
| franka_front | 720×1280 | 672×672 | 42 | 16×16 |

> **注意：** franka 相机原始分辨率不是正方形，但经 resize 后均为 672×672，然后以同样的总 stride=42 划分成 16×16 的 patch 网格。

## 3. Vision Tokenizer 详解

### 3.1 VisionTokenizer 架构

```python
class VisionTokenizer(nn.Module):
    def __init__(self, input_size):
        self.patch_stride = 14           # Conv2d 核和步长
        self.conv_stride = 3              # 局部注意力池化步长
        self.patch_embedding = nn.Conv2d(3, input_size, kernel_size=14, stride=14)
        self.class_embedding = nn.Parameter(...)  # 可学习的 [CLS] token
        self.split_embedding = nn.Parameter(...)
        self.local_attention = LocalAttention(input_size, conv_stride=3)
        self.global_attention = GlobalAttention(input_size)
```

### 3.2 Patch Embedding 计算

```
输入:  RGB图像 [B, 3, 672, 672]
Conv2d(3, hidden_size, kernel=14, stride=14):
输出:  [B, hidden_size, 48, 48]   # 672/14 = 48
```

然后经过 LocalAttention（步长3的池化）：

```
[48, 48] → avg_pool(stride=3) → [16, 16] = 256 patches
```

### 3.3 LocalAttention 模块

```python
class LocalAttention(nn.Module):
    # 将每个 patch 内的 3×3 像素块通过注意力机制聚合
    # Q: 来自下采样后的单点特征
    # KV: 来自原始 3×3 像素块
    # 输出: 保留了空间局部性的聚合特征 [B, hidden_size, 16, 16]
```

### 3.4 GlobalAttention 模块

```python
class GlobalAttention(nn.Module):
    # 使用可学习的 class_embedding 作为 Query
    # 对所有 256 个 patch tokens 做 Cross-Attention
    # 提取全局语义信息，注入到每个 patch token 中
```

### 3.5 Forward 流程图

```
pixel_values [B, 3, 672, 672]
    ↓
patch_embedding (Conv2d stride=14)
    ↓ patch_embeds [B, D, 48, 48]
    ↓
local_attention (stride=3 pooling + attention)
    ↓ patch_embed [B, D, 16, 16]
    ↓
class_embedding + global_attention → class_embed [1, D]
    ↓ class_embed broadcast and add to patch_embed
    ↓
modules(patch_embed)  ← 外接的 VAE / quantize 模块
    ↓
patch_embeds_ [list of B tensors, each [256, D]]
patch_hw_    [list of B tensors, each [2]]
```

## 4. 对比损失模块（Contrastive Losses）

### 4.1 SceneLevelContrastiveLoss

- **作用：** 整场景级别的 Image-PointCloud 对比
- **输入：** 所有 patch tokens 的平均池化向量
- **结构：** Linear → ReLU → Linear 投影头
- **损失：** 双向 InfoNCE（image→pc 和 pc→image）

### 4.2 CoordinateAwareContrastiveLoss

- **作用：** 坐标感知的 patch 级别对比
- **核心机制：** 根据 3D→2D 投影的 patch_idx，找到每个 3D 点对应的 image patch 特征
- **有效掩码：** 只对投影到图像范围内的点计算损失
- **损失：** 双向 InfoNCE

### 4.3 TactileContrastiveLoss

- **作用：** 触觉-视觉-点云 三模态对比
- **输入：** 触觉特征、点云特征、图像特征，以及正样本索引
- **损失：** Tac-PC 损失 + Tac-Image 损失的均值

## 5. 生成模块（MultimodalGenerationManager）

### 5.1 ImageGenerationModule

```
输入: LLM隐藏状态 + 当前图像patch特征
Query-based Intent Decoder → 提取高层意图特征
MAE Decoder (with ROI mask) → 生成patch级别的Delta/Alpha/Offset
最终输出: 修复后的图像patch序列
```

### 5.2 PointCloudGenerationModule

```
输入: LLM隐藏状态 + 当前点云
Feature Projector → 维数变换
FPS 采样 → 获取group中心点
Transformer Decoder → 预测每个group的future点云位移
输出: 预测的未来点云 (G×M, 3)
```

### 5.3 TactileGenerationModule

```
输入: LLM隐藏状态
Query-based Transformer Decoder → 生成触觉token
输出: 触觉特征向量
```

## 6. Tactile Tokenizer（触觉嵌入器）

### 6.1 核心发现：没有独立的 Tactile Tokenizer

项目中**没有独立的 tactile tokenizer 文件**，触觉数据通过 `tactile_embedder` 进行嵌入，这是 `ActionEmbedder` 的别名。本质上是一个 MLP（多层感知机），将低维触觉向量映射到高维 token 空间。

### 6.2 ActionEmbedder（tactile_embedder）结构

**定义位置：** `/media/sysu/MLA/MLA/models/diffusion/models.py`（第112行）

```python
class ActionEmbedder(nn.Module):
    def __init__(self, action_size, hidden_size):
        super().__init__()
        approx_gelu = lambda: nn.GELU(approximate="tanh")
        self.mlp = Mlp(
            in_features=action_size,        # 输入维度 = tactile_dim * n_arms
            hidden_features=hidden_size,   # 隐藏层维度 = token_size（模型维度）
            out_features=hidden_size,       # 输出维度 = token_size
            act_layer=approx_gelu,
            drop=0
        )

    def forward(self, x):
        return self.mlp(x)
```

> `Mlp` 来自 `timm.models.vision_transformer`，结构为：Linear → GELU → Linear

### 6.3 Tactile 数据处理流程（prismatic.py）

```
原始触觉数据 tactile [B, tactile_dim * n_arms]
    ↓
torch.chunk(t_flat, n_arms, dim=-1)  →  按机械臂数量切分
    ↓
每个触觉切片 [B, tactile_dim] → ActionEmbedder → [B, 1, token_size]
    ↓
torch.cat(tac_emb_list, dim=1)  →  [B, n_arms, token_size]  多臂触觉token
    ↓
与 fused_tokens（图像+点云token）拼接
    ↓
用于对比损失（TactileContrastiveLoss）和生成损失
```

### 6.4 关键参数

| 参数 | 值 | 说明 |
|------|----|------|
| `tactile_dim` | 12 或 24 | 触觉传感器维度（12=单臂，24=双臂） |
| `n_arms` | 从 `gripper_xyz` 推断 | 机械臂数量 |
| `hidden_size` | `token_size`（模型维度，如 4096） | 嵌入后token维度 |

### 6.5 Tactile 参与的两个任务

**① TactileContrastiveLoss（对比损失）**
- 触觉token vs 点云token：正样本通过最近邻匹配（CDist）
- 触觉token vs 图像token：正样本通过投影坐标索引匹配

**② TactileGenerationModule（生成任务）**
- 输入：LLM 隐藏状态
- 结构：Query-based Transformer Decoder（2层，4头）
- 输出：`(B, tactile_dim)` 预测的下一时刻触觉数据
- 损失：`F.mse_loss(tactile_pred, gt_tactile)`
