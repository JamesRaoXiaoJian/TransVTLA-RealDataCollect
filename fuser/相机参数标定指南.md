# 相机参数标定指南

> 项目路径：`/media/sysu/MLA/MLA/models/mla/fuser/camera.py`
> 文档目的：为 MLA 算法标定并填写 RealSense（机械臂腕部）和 DJI Action5（侧面）相机参数

---

## 一、算法需要哪些参数

算法使用**针孔相机模型**，需要以下三组参数：

### 1. 内参矩阵 K（Intrinsic Matrix）

```
K = [fx,  0, cx]
    [ 0, fy, cy]
    [ 0,  0,  1]
```

| 符号 | 含义 | 单位 | 典型范围 |
|------|------|------|---------|
| fx, fy | 焦距（像素） | pixel | 300~800 |
| cx, cy | 主点偏移 | pixel | 约为分辨率的一半 |

### 2. 旋转矩阵 R（Rotation Matrix）

- **尺寸：** 3×3，正交矩阵（R·Rᵀ = I，det(R) = 1）
- **含义：** 世界坐标系 → 相机坐标系的旋转变换
- **你的代码中的约定：** `R` 是 `world_to_cam`（世界→相机）

### 3. 平移向量 t（Translation Vector）

- **尺寸：** 3×1
- **含义：** 世界坐标系原点在相机坐标系中的位置
- **你的代码中的约定：** `t` 是相机光心在**世界坐标系**中的坐标
- 代码中实际使用时转换为：`t_world_to_cam = -R.T @ t`

## 二、两台相机的参数需求

### 相机 A：RealSense（机械臂腕部）

> 位置：安装在机械臂末端，随机械臂运动
> 型号：Intel RealSense D455（或类似）

| 参数 | 用途 | 获取方式 |
|------|------|---------|
| K | 3D→2D 投影（像素） | RealSense SDK 直接读取 |
| R, t | 世界→相机坐标变换 | **手眼标定（Hand-Eye Calibration）** |

### 相机 B：DJI Action5（侧面第三视角）

> 位置：固定安装，拍摄机器人侧面全景
> 型号：DJI Action 5 Pro

| 参数 | 用途 | 获取方式 |
|------|------|---------|
| K | 3D→2D 投影（像素） | **棋盘格/ChArUco 标定** |
| R, t | 世界→相机坐标变换 | **PnP 标定**（固定相机测基站坐标） |

### 3.2 RealSense 外参 R, t — 手眼标定

#### 方案 A：机械臂末端直接获取（推荐）

如果你的机械臂 API 可以直接输出**末端执行器在基座坐标系下的位姿**（齐次变换矩阵 4×4），这是最简单的方式：

```python
# 以 Franka 为例（ROS / libfranka）
import numpy as np

# 获取末端执行器到基座的变换矩阵（从机械臂 API）
# F_NE = 末端执行器在基座坐标系下的位姿
# F_CE = 相机在末端执行器坐标系下的位姿（手眼标定结果，待求）

# 手眼标定：求解 X，使 F_CE = F_NE⁻¹ · F_World · X
# 其中 F_World 是标定板在世界坐标系下的位姿
```

#### 方案 C：OpenCV 手眼标定（最直接）

```python
import numpy as np
import cv2

# ================================================================
# 1. 准备：采集 15~20 组数据
#    - 机械臂带动末端执行器（带 RealSense）运动到不同位姿
#    - AprilTag / ChArUco 标定板固定放置在相机视野内
#    - 每组记录：
#      (a) 机械臂末端在基座坐标系下的位姿 T_base_ee（从机器人 API 获取）
#      (b) 标定板在相机坐标系下的位姿 T_cam_target（从图像检测获取）
# ================================================================

T_base_ee_list = []   # 4x4 齐次变换矩阵
T_cam_target_list = [] # 4x4 齐次变换矩阵

# ================================================================
# 2. OpenCV 手眼标定
#    相机固定在末端（eye-on-hand）：AX = XB
# ================================================================

R_cam2ee, t_cam2ee = cv2.calibrateHandEye(
    R_gripper2base=[m[:3, :3] for m in T_base_ee_list],
    t_gripper2base=[m[:3, 3] for m in T_base_ee_list],
    R_target2cam=[m[:3, :3].T for m in T_cam_target_list],  # 转置得 cam_to_target
    t_target2cam=[-m[:3, :3].T @ m[:3, 3] for m in T_cam_target_list],
    method=cv2.CALIB_HAND_EYE_TORSSI
)

# R_cam2ee, t_cam2ee = 相机在末端执行器坐标系下的位姿

# ================================================================
# 3. 合并：得到相机在基座坐标系下的位姿
#    T_base_cam = T_base_ee · T_cam2ee
# ================================================================

T_base_cam = T_base_ee @ np.block([[R_cam2ee, t_cam2ee], [0, 0, 0, 1]])

R = T_base_cam[:3, :3]   # 世界(基座) → 相机
t = T_base_cam[:3, 3]     # 相机光心在基座坐标系中的坐标

print(f"R = \n{R}")
print(f"t = {t}")
```

> **注意：** 代码中使用的是 `R_world_to_cam`（世界→相机）和 `t`（相机在世界中的坐标）。如果你的标定结果是相机→基座，需要取逆（R 取转置，t 取负）。

### 3.3 DJI Action5 内参 K — 棋盘格标定

```python
import cv2
import numpy as np
import glob

# ================================================================
# 1. 打印棋盘格标定板（建议 9x7 或 11x8 格，内角点）
# ================================================================

CHECKERBOARD = (9, 6)  # 内角点数
SQUARE_SIZE = 0.040    # 棋盘格方格尺寸（米）

objp = np.zeros((CHECKERBOARD[0] * CHECKERBOARD[1], 3), np.float32)
objp[:, :2] = np.mgrid[0:CHECKERBOARD[0], 0:CHECKERBOARD[1]].T.reshape(-1, 2)
objp *= SQUARE_SIZE

obj_points, img_points = [], []

# ================================================================
# 2. 拍摄 20~30 张不同角度的标定照片
#    （确保覆盖画面中心和四个角）
# ================================================================

images = glob.glob('calib_imgs/*.jpg')
for fname in images:
    img = cv2.imread(fname)
    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

    ret, corners = cv2.findChessboardCorners(gray, CHECKERBOARD, None)
    if ret:
        # 亚像素精度优化
        criteria = (cv2.TERM_CRITERIA_EPS + cv2.TERM_CRITERIA_MAX_ITER, 30, 0.001)
        corners = cv2.cornerSubPix(gray, corners, (11, 11), (-1, -1), criteria)
        obj_points.append(objp)
        img_points.append(corners)

# ================================================================
# 3. 标定
# ================================================================

ret, mtx, dist, rvecs, tvecs = cv2.calibrateCamera(
    obj_points, img_points, gray.shape[::-1], None, None
)

print(f"K = \n{mtx}")
print(f"畸变系数 dist = {dist.ravel()}")

# 畸变较小可以忽略（Action5 电子防抖+畸变校正后）
# 如果想校正畸变：
# img_undistorted = cv2.undistort(img, mtx, dist, None, mtx)
```

> **分辨率建议：** DJI Action5 拍摄时固定为 4:3 或 16:9，分辨率 4000×3000 或 3840×2160。标定时用实际工作的分辨率。

### 3.4 DJI Action5 外参 R, t — PnP 标定

#### 方法 1：人工测量（最简单，精度要求不高时推荐）

1. 在机器人基座上选一个参考点（标定针尖），用机械臂末端抵近
2. 测量相机光心到机器人基座原点的物理距离（卷尺）
3. 用量角器或相机视角估算相机朝向

```
t = [Δx, Δy, Δz]  （米，单位需转为与3D点云一致）
R = 从朝向角度构造旋转矩阵
```

#### 方法 2：PnP 联合标定（更精确）

```python
# ================================================================
# 1. 准备：放一个 Apriltag 或 ChArUco 标定板在工作空间
#    测量标定板在机器人基座坐标系下的 3D 坐标（人工用卷尺）
# ================================================================

# 标定板角点在基座坐标系下的 3D 坐标（人工测量）
objp_world = np.array([...], dtype=np.float32)  # shape: (N, 3)

# ================================================================
# 2. 拍摄多帧标定板图像，检测像素坐标
# ================================================================

# 每帧同时记录：标定板 3D 坐标 + 像素坐标
img_points_list = []
obj_points_list = []

for i, img in enumerate(calib_images):
    corners, ids = detect_apriltag(img)  # 用 apriltag_ros 或 apriltag Python
    img_points_list.append(corners)       # 像素坐标 (N, 2)
    obj_points_list.append(objp_world)     # 对应的 3D 坐标 (N, 3)

# ================================================================
# 3. 用 solvePnP 求每帧的相机外参
# ================================================================

# 取均值作为相机外参
R_cum, t_cum = [], []
for obj3d, img2d in zip(obj_points_list, img_points_list):
    _, rvec, tvec = cv2.solvePnP(
        obj3d, img2d, K_DJI, dist_coeffs,
        flags=cv2.SOLVEPNP_EPNP
    )
    R_cam, _ = cv2.Rodrigues(rvec)  # 旋转向量 → 矩阵
    R_cum.append(R_cam)
    t_cum.append(tvec)

R_avg = np.mean(R_cum, axis=0)
t_avg = np.mean(t_cum, axis=0)

# 注意：solvePnP 返回的是
#   世界坐标系点在相机坐标系下的表示
#   即：P_cam = R · P_world + t
# 对应代码中的 world_to_camera：R_world_to_cam = R_avg, t_world_to_cam = t_avg

R = R_avg
t = t_avg.ravel()
```

### 4.2 步骤 2：在 `contrastive.py` 中添加投影函数

```python
# 文件：/media/sysu/MLA/MLA/models/mla/fuser/contrastive.py

def project_3d_to_2d_672_dji_action_side(
    xyz_3d,
    K: torch.Tensor,
    R: torch.Tensor,
    t: torch.Tensor,
    image_size_orig: tuple = (2160, 3840),  # DJI Action5 实际分辨率
    image_size_resize: tuple = (672, 672),
    vision_strides: dict = {"patch_stride": 14, "conv_stride": 3},
):
    scale_x = image_size_resize[1] / image_size_orig[1]
    scale_y = image_size_resize[0] / image_size_orig[0]

    K_scaled = K.clone()
    K_scaled[0, 0] *= scale_x
    K_scaled[1, 1] *= scale_y
    K_scaled[0, 2] *= scale_x
    K_scaled[1, 2] *= scale_y

    R_world_to_cam = R.T
    t_world_to_cam = -R_world_to_cam @ t

    xyz_cam = xyz_3d @ R_world_to_cam.T + t_world_to_cam
    uvw = xyz_cam @ K_scaled.T
    z = uvw[..., 2:]
    xy = uvw[..., :2] / (z + 1e-6)
    total_stride = vision_strides["patch_stride"] * vision_strides["conv_stride"]

    row = (xy[..., 1] / total_stride).floor().long()
    col = (xy[..., 0] / total_stride).floor().long()

    patch_h = image_size_resize[0] // total_stride
    patch_w = image_size_resize[1] // total_stride

    valid = (z.squeeze(-1) > 0) & \
            (xy[..., 0] >= 0) & (xy[..., 0] < image_size_resize[1]) & \
            (xy[..., 1] >= 0) & (xy[..., 1] < image_size_resize[0])

    row = torch.clamp(row, 0, patch_h - 1)
    col = torch.clamp(col, 0, patch_w - 1)
    patch_idx = torch.stack([row, col], dim=-1)

    return patch_idx, valid
```

## 五、R 和 t 的符号约定（重要）

代码中的使用方式：

```python
R_world_to_cam = R.T      # 世界→相机 = R 的转置（逆）
t_world_to_cam = -R.T @ t # 世界→相机 平移
```

这意味着：如果你通过标定得到的 T_cam_to_world（相机→世界），需要取逆：

```
R_world_to_cam = R_cam_to_world.T
t_world_to_cam = -R_cam_to_world.T @ t_cam_to_world
```

**填写时的检查：** 用已知的 3D 点验证投影是否正确，例如：

```python
# 世界原点 (0, 0, 0) 投影到相机平面应该在光心附近
# 如果填写方向错误，投影点会跑到图像外
```

## 七、注意事项

1. **RealSense 已经是你的 `franka_right`**，内参（fx=387.4, fy=386.9）已填写，只需要做**手眼标定**获取外参 R, t
2. **DJI Action5** 需要先确定实际工作分辨率（4K 还是 1080p），标定时用相同分辨率
3. `vision_strides` 的 `conv_stride` 决定局部注意力池化的 stride，你的 vision_tokenizer 中默认是 3
4. 两台相机的 `image_size_resize` 都会被 resize 到 672×672，但原始分辨率和内参需要是真实值（代码会按比例缩放 K）
